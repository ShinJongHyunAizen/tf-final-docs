# FO-LOG-PAG-001 — 로그인 계정 잠금

## 공통 전제 — 무차별대입 대응 처리

무차별대입을 방지하기 위해 아래 조건인 경우 계정 잠금 처리:

| 케이스 | 조건 |
|--------|------|
| a | 동일 세션에서 1분 내 로그인 시도를 10회 이상 시도하여 실패한 경우 |
| c | 동일 아이디로 비밀번호를 10회 이상 틀린 경우 |

---

## Description 1 — 로그인 잠금 케이스

### 계정을 무작위로 틀렸을 때 (케이스 a, b)

| 케이스 | 조건 | 처리 |
|--------|------|------|
| a | 동일 세션에서 1분 내 로그인 시도 **15회** 이상 실패 | **10분**간 로그인 불가 처리 |
| a 예외 | 다른 세션에서 시도 | 로그인 가능 |
| b | 서버를 통해 1분 내 로그인 시도 **15회** 이상 실패 | **10분**간 로그인 불가 처리 |

### 동일 아이디를 계속 틀렸을 때 (케이스 c)

| 조건 | 처리 |
|------|------|
| 동일 아이디로 비밀번호 **15회** 이상 틀린 경우 | **10분**간 로그인 불가 처리 |

**추가 규칙:**
- **동일 아이디라면 각각 다른 세션에서 10번 시도를 하더라도 총 시도 횟수가 15번이면 잠금 처리**
- **동일 아이디라면 이미 잠금 처리된 이후 다른 세션에서 시도하더라도 잠금 처리 유지**

---

## Description 2 — 로그인 잠금 얼럿창

### 노출 조건
- 10회째 계정 확인 시도 시 계정이 틀림이 확인되어 계정잠김 리턴값을 받으므로 **10번째에 로그인 잠금 얼럿창 노출**
- `[로그인]` 버튼 클릭 시 로그인 잠금 얼럿창 노출

### 얼럿 문구
```
아이디 또는 비밀번호 입력이 10회 이상 틀려 계정이 보호(잠금) 처리되었습니다.
안전한 서비스 이용을 위해 5분 후 로그인을 다시 시도할 수 있습니다.
잠시 기다려 주시기 바랍니다.

남은 시간 : 04:59
```

### 버튼

| 버튼 | 동작 |
|------|------|
| `[확인]` | 얼럿창 닫힘 |

### 남은 시간 카운팅 표기
- 표기 방법: `05:00` 으로부터 남은 시간 카운팅 처리 `HH:MM`
- **카운팅 시작 후 `[확인]` 을 눌러 얼럿창이 꺼지고 다시 `[로그인]` 을 시도한 경우 → 카운팅은 계속해서 진행**되도록 처리
- 잠금 이후 **5분간 로그인 불가** → 5분 경과 시 로그인 가능
