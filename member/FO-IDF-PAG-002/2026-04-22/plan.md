# FO-IDF-PAG-002 — 아이디 찾기 완료 구현 플랜

## 전제 조건
- FO-IDF-PAG-001 (아이디 찾기) 본인인증 성공 후 진입
- 회원 유형(대표/직원) 및 사업자명 정보를 아이디 찾기 응답에 포함

---

## PLAN-01: 아이디 찾기 완료 응답 데이터 구성

아이디 찾기 API 응답에 회원 유형, 아이디, 사업자명(대표인 경우)을 포함한다.
직원인 경우 사업자명 없이 아이디만 반환한다.
사업자명 마스킹 처리는 프론트엔드에서 수행한다.

참조 섹션: Description 1 — 아이디 노출 규칙

예상 파일:
- `backend/src/main/java/com/example/application/dto/IdFindResponse.java`
- `backend/src/main/java/com/example/application/service/MemberIdFindService.java`

테스트 포인트:
- 대표 회원 조회 시 아이디 + 사업자명 포함 응답 확인
- 직원 회원 조회 시 아이디만 포함 응답 확인 (사업자명 null 또는 미포함)

---

## PLAN-02: 사업자명 마스킹 처리 로직

사업자명 글자수에 따라 마스킹 규칙을 적용한다.
- 7자 미만: 앞 2자리 노출 + 나머지 글자수만큼 `*` 마스킹
- 7자 이상: 앞 6자리 노출 + 나머지 글자수만큼 `*` 마스킹

참조 섹션: Description 1 — 아이디 노출 규칙 (대표 → 사업자명 마스킹 처리)

예상 파일:
- `frontend/src/utils/masking.ts`

테스트 포인트:
- 사업자명 6자(`가나다라마바`) → `가나****` 노출 확인 (7자 미만: 앞 2자리 + 4자 마스킹)
- 사업자명 7자(`가나다라마바사`) → `가나다라마바*` 노출 확인 (7자 이상: 앞 6자리 + 1자 마스킹)
- 사업자명 10자 → 앞 6자리 노출 + 나머지 4자 마스킹 확인

---

## PLAN-03: 아이디 찾기 완료 화면 UI

안내 문구 `아이디를 찾았어요.` 를 표시하고, 회원 유형에 따라 아이디와 사업자명(마스킹 처리)을 노출한다.
직원인 경우 사업자명 영역을 미노출한다.
`[로그인]`과 `[비밀번호 찾기]` 버튼을 표시한다.

참조 섹션: Description 1 — 안내 문구, 아이디 노출 규칙, 버튼

예상 파일:
- `frontend/src/features/member/components/IdFindResultScreen.tsx`
- `frontend/src/pages/IdFindCompletePage.tsx`

테스트 포인트:
- 안내 문구 `아이디를 찾았어요.` 노출 확인
- 대표 → 아이디 + 마스킹된 사업자명 노출 확인
- 직원 → 아이디만 노출, 사업자명 영역 미노출 확인
- `[로그인]` 클릭 → FO-LOG-PAG-001 이동 확인
- `[비밀번호 찾기]` 클릭 → FO-PWF-PAG-001 이동 확인

---

## PLAN-04: 통합 테스트

참조 섹션: Description 1 전체

테스트 포인트:
- 대표 회원: 아이디 찾기 완료 → 사업자명 마스킹 규칙(7자 미만/이상) 각각 확인
- 직원 회원: 아이디 찾기 완료 → 아이디만 노출 확인
- `[로그인]` → 로그인 화면 이동 확인
- `[비밀번호 찾기]` → 비밀번호 찾기 화면 이동 확인
