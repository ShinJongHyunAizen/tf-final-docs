# FO-REG-PAG-001 — 회원가입

## Description 1 — 사업자 정보 인증 및 회원가입 폼

### 화면 안내 타이틀
```
사업자 정보를 인증해 주세요.
사업자등록번호가 있는 사업자에 한해
회원가입이 가능합니다.
```

### 입력 필드 구성

| 필드 | 플레이스홀더 / 안내 | 비고 |
|------|------|------|
| 사업자 번호 | 사업자 번호 (- 없이 숫자 10자리 입력) | 에러: 사업자 번호를 '-'없이 숫자 10자리를 입력해 주세요. |
| 설립일 | - | - |
| 상호명 | - | 도움말: 사업자명은 사업자등록증의 전체 사업자명을 입력해 주세요. |
| 대표자 | - | 도움말 (#CCNLND-3454): 공동 대표인 경우, 첫 번째로 기재된 대표자 이름만 입력해 주세요. |

### 사업자 정보 인증 완료 후 노출 문구
```
{사업자명} 님
사업자 정보 인증이 완료되었어요.
회원가입을 진행해 주세요.
```

### 사업자 구분 (인증 후 변경 불가)
사업자 번호 가운데 2자리 기준으로 자동 분류, 인증 후 고정 노출:

| 가운데 2자리 | 구분 |
|---|---|
| 01~79, 80, 89, 90~99 | 개인사업자 |
| 81~87 | 법인사업자 |

### 아이디 중복 검사 (#CCNLND-1895)

- 아이디 입력 후 필드를 벗어나는 경우 → 아이디 중복확인 영역 노출
- 안내 텍스트: `아이디 중복 여부를 확인해 주세요.`

| 버튼 | 동작 |
|------|------|
| `[아이디 중복 확인]` | 아이디 중복값 체크 |

**아이디 중복 기준: 대소문자 구분하지 않음 (#CCNLND-2585)**
- 예) `@WaregeT02` 와 `@wAregeT02` 는 동일하다고 본다

#### 중복 검사 결과 분기

```
중복 확인 버튼 클릭
├── 중복아님 (true)
│   ├── 얼럿: 사용 가능한 아이디 입니다.
│   └── 3번 영역(중복확인 영역) 사라짐
│       └── 다시 아이디 영역 입력 후 필드 벗어나면 → 중복확인 영역 재노출
└── 중복 (false)
    ├── 얼럿: 가입이 불가한 아이디입니다.\n다른 아이디로 시도해 주세요.
    └── 3번 영역 그대로 유지
```

### 도움말 문구

- 필드 에러메시지가 노출될 경우, 도움말 문구는 에러메시지 위로 노출
- 사업자명 하단: `사업자명은 사업자등록증의 전체 사업자명을 입력해 주세요.`
- 대표자 하단 (#CCNLND-3454): `공동 대표인 경우, 첫 번째로 기재된 대표자 이름만 입력해 주세요.`

### 비밀번호 처리

- 비밀번호 재확인 필드 삭제
- default: 입력 시 마스킹 처리, 비밀번호 뷰 off 아이콘
- 비밀번호 뷰 아이콘 클릭 시 → 입력한 비밀번호 노출, 뷰 on 아이콘으로 변경
- 비밀번호 뷰 on 아이콘 재클릭 시 → 비밀번호 마스킹 처리, 뷰 off 아이콘 노출
