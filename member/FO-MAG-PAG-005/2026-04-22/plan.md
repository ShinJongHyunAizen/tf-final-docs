## PLAN-01: 탈퇴 사유 입력 UI

탈퇴 사유 선택 목록과 탈퇴 유의사항, 동의 체크박스를 표시한다.
'기타' 선택 시 입력 필드를 노출하고, 입력값이 없으면 `[회원 탈퇴]` 버튼 클릭 시 얼럿을 표시한다.
버튼 활성화 조건은 탈퇴 사유 선택 + 유의사항 동의 체크박스 체크 모두 충족해야 한다.

참조 섹션: Description 1 — 1. 회원탈퇴 사유 선택, 2. 회원 탈퇴 전 유의사항, 3. 회원 탈퇴 버튼

예상 파일:
- `frontend/src/features/member/components/WithdrawalForm.tsx`
- `frontend/src/pages/WithdrawalPage.tsx`

테스트 포인트:
- 탈퇴 사유 5개 선택지 노출 확인: 대출 조건 및 비용 불만족 / 서비스 품질 문제 / 보안 또는 개인정보 문제 / 서비스 불필요 / 기타
- '기타' 선택 시 입력 필드 노출, 플레이스홀더 `기타 탈퇴 사유를 입력해 주세요.(30자 이내)` 확인
- 사유 미선택 또는 체크박스 미체크 시 버튼 비활성화 확인
- '기타' 선택 + 사유 미입력 + 버튼 클릭 시 얼럿 `기타 사유를 입력해 주세요.` 노출 확인
- 유의사항 문구 노출 확인:
  ```
  회원 탈퇴 시 회원 정보와 서비스 이용 기록은 모두 삭제되며 삭제된 데이터는 복구할 수 없습니다.
  다만 법령 또는 회사 내부 정책에 따라 보관해야 하는 정보는 탈퇴 후에도 일정 기간 보관됩니다.
  자세한 사항은 개인정보 처리 방침에서 확인하실 수 있습니다.
  회원 탈퇴 후 동일 사업자 번호로 재가입 하더라도 탈퇴 전 회원정보와 서비스 이용 기록은 복구되지 않습니다.
  ```
- 동의 체크박스 문구: `탈퇴 유의사항을 확인하였으며 탈퇴에 동의합니다.` 확인

## PLAN-02: 탈퇴 가부 판단 API (대표 회원 5단계 플로우)

`[회원 탈퇴]` 버튼 클릭 시 서버에서 대표 회원의 탈퇴 가부를 판단한다.
직원은 조건 없이 탈퇴 가능하며, 대표는 아래 5단계 순서로 판단한다.
1. 진행중인 데이터제공동의 미확인 & 만료일 미경과 → 탈퇴 불가
2. 대출 상태 == 신청완료 → 탈퇴 불가
3. 데이터 이용 상태 이용중 (대출 및 데이터제공) → 탈퇴 불가
4. 현재 진행중인 데이터 이용건 있음 → 탈퇴 불가
5. 등록 직원 있음 → 탈퇴 불가
6. 조건 없음 → 탈퇴 진행

참조 섹션: Description 2 — 회원 유형별 탈퇴 조건, 대표 탈퇴 가부 판단 플로우

예상 파일:
- `backend/src/main/java/com/example/application/service/WithdrawalCheckService.java`
- `backend/src/main/java/com/example/adapter/in/web/WithdrawalController.java`
- `backend/src/main/java/com/example/application/dto/WithdrawalCheckResponse.java`

테스트 포인트:
- 직원 → 예외조건 없이 탈퇴 가능 응답 확인 (#CCNLND-2726)
- 대표 → 각 5단계 조건별 탈퇴 불가 응답 확인
- 조건 없음 → 탈퇴 진행 응답 확인

확인필요: [대출 & 데이터 이용중 동시 존재 시 우선순위 — 2번과 3번 얼럿 중 어느 쪽이 먼저 노출되는지]
